SQL

Le SQL, ou “Structured Query Language” (Langage de Requêtes Structurées en français) est un langage permettant de créer et gérer des bases de données. Le SQL est un langage très populaire, et a été décliné sous plusieurs versions.

Parmi ces versions, j’ai eu particulièrement l’occasion de travailler sur MySQL, SQLite, et HFSQL, dont voici les spécifications :

  • MySQL est une version extrêmement versatile de SQL. Nous pouvons l’employer pour des applications Linux, MacOS, Windows, ou encore web. Elle est également très populaire, utilisée par un grand nombre de sites et d’applications. Nous pouvons en trouver deux types de licence : gratuite et payante ; bien que je n’aie travaillé que sur la licence gratuite. C’est à travers cette version que j’ai appris les bases du langage SQL, en le découvrant lors de ma première année d’études, afin de réaliser un site internet. Je le réemploierai deux ans plus tard, lors de ma troisième année d’études, afin de réaliser une application bien plus complète, du nom de Phytolearning.
  • SQLite est une version très légère de SQL. Elle est surtout employée pour les applications mobiles, où l’espace de stockage est beaucoup plus limité. Cette version est open source. J’ai particulièrement eu l’occasion de travailler avec cet outil lors de ma seconde année, afin de réaliser une application Android, nommée Hermès.
  • HFSQL est une version modifiée de SQL pour répondre aux besoins spécifiques d’un type de langage de programmation particulier, nommé le WLangage, mis au point par la société montpellieraine PC Soft. Contrairement aux deux autres versions citées plus haut, il s’agit d’une version propriétaire appartenant à PC Soft. C’est une technologie avec laquelle je me suis particulièrement familiarisée lors de mon arrivée en entreprise, en travaillant sur le projet Portail R&R.
